# How to get this live at `okayayushhh.github.io/portfolio`

Do this once. Takes ~5 minutes. You'll get a public URL you can paste into your assignment.

## 1. Make a repo on GitHub

1. Go to https://github.com/new
2. **Repository name:** `portfolio`  ← must be exactly this
3. Set it to **Public** (required for free GitHub Pages)
4. Leave everything else unchecked. Click **Create repository**.

## 2. Upload the files

On the empty repo page, click **uploading an existing file** (it's a link in the middle of the page).

Drag the **contents** of this folder into the upload box — not the folder itself, the things inside it:

- `index.html`
- `README.md`
- the whole `assets/` folder (GitHub keeps the folder structure when you drag it)

Scroll down, click the green **Commit changes** button.

## 3. Turn on GitHub Pages

1. In the repo, click **Settings** (top right)
2. In the left sidebar, click **Pages**
3. Under **Source**, pick **Deploy from a branch**
4. Under **Branch**, pick `main` and `/ (root)`. Click **Save**.

Wait 1–2 minutes. Refresh the Pages settings page. You'll see a green banner:

> Your site is live at **https://okayayushhh.github.io/portfolio/**

That's the link you paste into your assignment submission.

## 4. Making changes later

Any file you edit through GitHub's web UI (pencil icon → edit → commit) goes live in about 30 seconds. Same for drag-and-drop re-uploads.

---

**Tip:** if the photo doesn't show up on the live site, it's almost always because the `assets/` folder didn't get uploaded — not the file inside it, the folder itself. Re-upload and make sure the repo's file list shows both `index.html` and an `assets/` folder next to it.
